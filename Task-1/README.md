# Система бронювання квитків
## Опис предметної області
Система бронювання квитків надає можливість користувачам переглядати доступні рейси, вибирати місця та оформляти бронювання. Користувач може створювати та переглядати власні бронювання, а адміністратор має можливість додавати, редагувати та видаляти рейси. Після створення бронювання система генерує квиток і реєструє оплату.

## Архітектурний підхід
Система побудована за трирівневою layered архітектурою: Presentation (UI), Business Logic та Data Access. UI приймає дії користувача. Business Logic перевіряє доступність місць, генерує квитки, здійснює оплату. І Data Access відповідає за зберігання та читання даних (дані користувачів, рейси, квитки, оплати). Такий розподіл зменшує зв'язанність та полегшує тестування і підтримку.
